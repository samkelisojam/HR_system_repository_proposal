Project Overview

This project involves the development of an Employee Onboarding and Management System using a microservice architecture. The project will span 6 months and involve 7 developers, each responsible for two microservices, with additional roles for testing and security.

This system aims to streamline the onboarding process, improve employee management, and enhance overall operational efficiency. By utilizing microservices, we ensure scalability and maintainability, allowing for easier updates and integration of new features in the future.

We will be using .NET Web API for backend development and GitHub for version control and collaboration.

User Roles in the System

  • Project Leader: Oversees the project and coordinates between teams.
  • Developers: Responsible for building and maintaining microservices.
  • QA Engineers: Conduct testing to ensure quality and functionality.
  • Security Lead: Ensures that security measures are implemented.
  • DevOps Engineer: Manages deployment and CI/CD processes.

Project Plan

The project will follow a 6-month sprint duration. Each sprint will focus on specific milestones and deliverables, ensuring the timely completion of the project.

  • Sprint 1: Initial Setup and Requirement Analysis
  • Sprint 2: Design and Architecture
  • Sprint 3: Development of Core Microservices
  • Sprint 4: Integration and Testing
  • Sprint 5: Final Adjustments and Security Enhancements
  • Sprint 6: Deployment and Documentation
